# System Patterns

## Architecture
- `umalator` (JP) relies on `uma-skill-tools` for data and logic.
- `umalator-global` appears to be a standalone or forked version with its own data files (`skill_data.json`).

## Data Flow
- Skill definitions are stored in JSON files.
- `uma-skill-tools/data/skill_data.json`: Master list for JP.
- `umalator-global/skill_data.json`: Master list for Global.

## Key Files
- `uma-skill-tools/data/skill_data.json`
- `umalator-global/skill_data.json`
